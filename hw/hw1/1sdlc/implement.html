The Implementation stage is where we get to work, after we have gathered as much
information as we can about the task ahead of us. There could be physical elements
involved in the process. We will certainly be writing code at this point.
Still, even with the planning stage behind us, we should treat every design doc as
living document that will change over time. Last minute considerations always pop
up - then we start hearing the word "agile" over and over again in meetings.

Do you like to work overtime for free, because you are paid a salary, and not by the
hour? Well you do now.

We haven't discussed testing yet, but during this phase we will certainly be
testing our code.
