Test and Deploy

Main Plan Implement Test & Deploy
According to Techopedia, Testing is the process of finding flaws and defects
in created in software. (it is also a great way to make enemies within
the dev team.)

Documentation is a key element in the testing effort, as it is vital to keep
Track of the overall state of the product.

Once the software is in a stable enough state, and all the necessary
stakeholders have approved, the software is deployed (sold to customers).

After deployment, there is a maintenance phase where bugs found after
deployment are addressed.
